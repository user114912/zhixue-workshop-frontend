## 代码规范

### 基本要求
- Python 代码遵循 PEP 8
- JavaScript 代码保持现有项目风格，不随意引入新框架
- 使用中文注释和文档字符串
- 要有完整的项目记录文档（project_plan.md），修改代码，创建文件，删除文件后要立刻修改，做任务前要读取一下
- 提交信息用中文描述
- 不生成无用的僵尸文档，文件，不需要的立马删除，同步更新文档(project_plan.md)
- 不要回答太乐观，实事求是
- 不要擅自删除、覆盖、回滚用户已有代码
- 删除文件前先确认

## 开发时多Agent协作系统

**系统概述**：在 VS Code 中使用 Claude Code 开发时，内置三个专业 Agent 角色自动运行，无需手动触发。

### 三个Agent（自动运行）

| Agent | 职责 | 触发时机 |
|-------|------|----------|
| **监督Agent** | 代码审查、规范检查、比赛要求验证 | 每次创建/修改/删除文件前，自动审查并报告计划 |
| **测试Agent** | 按需测试、效果验证、回归测试 | 每次创建/修改文件后，按项目实际情况验证 |
| **记忆Agent** | 全局记忆维护、变更记录、上下文保持 | 每完成 3 个 Step 后，自动更新 project_plan.md |

### 自动规则

1. **监督Agent**：你不需要手动叫 `/guardian`。每次要创建/修改/删除文件前，Claude Code 自动先输出一份影响分析："将修改 X 文件，影响 Y 功能，确认继续？"
2. **测试Agent**：你不需要手动叫 `/tester`。每次文件变更后，Claude Code 根据项目实际运行方式检查是否无报错；如果没有构建命令，要说明无法运行构建验证。
3. **记忆Agent**：你不需要手动叫 `/memory`。每完成 project_plan.md 中的 Step，Claude Code 自动更新 project_plan.md 的进度状态，测试记录不要覆盖之前记录的记录，保持可溯源性。
