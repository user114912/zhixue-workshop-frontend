Space is the **read / review** counterpart to the active surfaces. Where Chat / Co-Writer / Book are where you *produce*, Space is where everything you produce lives — searchable and replayable.

## What lives in Space

Space bundles four sub-surfaces under one navigation:

| Sub-surface | What's in it |
|-------------|--------------|
| **Chat History** | Every conversation across every mode (Chat / Solve / Quiz / Research / Visualize) |
| **Notebooks** | Categorised, colour-coded notebooks holding records from Chat, Research, and Co-Writer |
| **Question Bank** | Every auto-generated quiz question — bookmarkable, @-mention-able in chat |
| **Skills** | User-authored `SKILL.md` files that define teaching personas |

## Chat History

Every conversation, with:

- **Title rename** — edit the auto-generated title to something searchable
- **Delete** — drop a session (also removes its memory traces)
- **Resume** — open the session in a new tab or reattach to the active tab
- **Per-turn delete** — supported on every entry point (chat, solve, quiz, research, visualize)

The browser search box at the top filters across all titles and bodies — useful when you can't remember which session covered something.

## Notebooks

Notebooks are the **persistent storage for outputs you want to keep**. A notebook is:

- A name + description + color tag
- A list of records (each record is a Markdown document)
- Cross-linked back to the originating chat session / Co-Writer document

### Saving to a notebook

Most surfaces have a **Save to Notebook** action:

| Surface | What gets saved |
|---------|-----------------|
| **Chat** *(any mode)* | The current turn + its citations |
| **Co-Writer** | The current document |
| **Quiz** | The question + your answer + grading result |
| **Research** | The full report + outline + citations |

### Using a notebook in chat

Notebooks are queryable from chat via the `list_notebook` and `write_note` tools (auto-mounted when ≥1 notebook exists). You can also attach specific records to a turn via the composer's Space picker.

### CLI mirror

```bash
deeptutor notebook list
deeptutor notebook create research-notes --description "Daily reading log"
deeptutor notebook show research-notes
deeptutor notebook add-md research-notes ./paper-summary.md
deeptutor notebook replace-md research-notes <record_id> ./updated.md
deeptutor notebook remove-record research-notes <record_id>
```

## Question Bank

Every auto-validated quiz question generated by **Quiz** mode lands here:

- **Bookmark** — flag a question for later review
- **Categories** — questions inherit a topic tag based on the source
- **@-mention in chat** — reference past questions in a new chat turn ("@question 47, why is option B correct?")

The Question Bank is how spaced repetition becomes a real workflow: regenerate yesterday's wrong answers, get explanations, re-quiz.

## Skills

Skills are **user-authored persona definitions** — `SKILL.md` files following a specific schema. When a skill is active:

- Its persona is **injected into the chat system prompt** for the duration
- The skill's triggers can auto-activate it (e.g., when you mention specific topics)

### Skill schema

```markdown
---
name: socratic-tutor
description: Teach by asking probing questions instead of giving direct answers
triggers:
  - "explain"
  - "what is"
  - "teach me"
---

You are a Socratic tutor. Instead of giving direct answers:

1. Ask probing questions that reveal what the student already knows.
2. Guide them to discover the answer themselves.
3. Confirm understanding with a follow-up question.

Use clear, encouraging language. Never lecture.
```

DeepTutor ships built-in archetypes — Socratic, encouraging, rigorous — and you can author your own from the Skills tab.

### Where skills live

- **Personal skills** — `data/user/workspace/skills/*.md`
- **Project-scoped skills** — `.deeptutor/skills/*.md` *(if you're using a project workspace)*

## Pairs well with

- **Chat** — Pull a notebook record / past session into the current turn for context
- **TutorBot** — Bots can also use Skills (per-bot configuration)
- **Memory** — Notebooks become L1 traces in the `notebook` surface, eventually consolidated into L3
